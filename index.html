Sales Workflow Management System

ระบบจัดการงานขาย เอกสาร และ Approval Flow สำหรับ บริษัท กรีนส์ดี จำกัด (GREEN Dii Co., Ltd.)

🎯 Flow Document for Business Presentation

ภาพรวมระบบ (System Overview)

ระบบนี้ออกแบบมาเพื่อ Digitalize กระบวนการทำงานของฝ่ายขาย ตั้งแต่การสำรวจหน้างาน การทำใบเสนอราคา การเปิด Work Order จนถึงการอนุมัติเอกสารและส่งงานให้จัดซื้อ/โรงงาน ครอบคลุม 4 เอกสารหลัก:

📋 Quotation (ใบเสนอราคา)

สร้างใบเสนอราคาพร้อมเลขอัตโนมัติตามเซลล์ ส่ง PDF ทางเมล์ให้ลูกค้า พร้อมแนบแบบ (Drawing)

QGD-0326-KC001

03=เดือน, 26=ปี, KC=ชื่อย่อเซลล์, 001=ลำดับ

🔧 Work Order (ใบสั่งงาน)

เปิดใบสั่งงานสำหรับโครงการ มีรายละเอียดสินค้า ทีมงาน และเอกสารเช็คลิสต์

WO26001

26=ปี, 001=ลำดับรันต่อเนื่อง (ไม่แบ่งตามเซลล์)

🤝 Hand Over Job (ใบส่งมอบงาน)

เอกสารส่งมอบงาน พร้อมแบบประเมินคุณภาพจากลูกค้า

HO26001

26=ปี, 001=ลำดับรันต่อเนื่อง (ไม่แบ่งตามเซลล์)

🛒 Purchase Request (ใบขอซื้อ)

ใบขอซื้อสำหรับสั่งของต่างประเทศ มีรายละเอียดราคาและ VAT

PR26001

26=ปี, 001=ลำดับรันต่อเนื่อง (ไม่แบ่งตามเซลล์)

ขั้นตอนการทำงาน (Workflow Steps)

👤 เซลล์ (Sales)

สำรวจหน้างาน (Site Survey)

เซลล์เดินทางไปสำรวจหน้างาน เก็บข้อมูลโครงการ วัดขนาด ถ่ายรูป และรวบรวมความต้องการของลูกค้า

👤 เซลล์ (Sales) 📧 Email

ส่งเมล์ขอ Min (Request Minimum Spec & Drawing)

เซลล์ส่งรายละเอียดงานทางเมล์เพื่อขอ Min (Minimum Specification) กรณีงานต้องมีแบบ ต้องขอแบบ (Drawing) ด้วย

  • รายละเอียดสินค้าและขนาด
  • จำนวนที่ต้องการ
  • แบบ (Drawing) - ถ้าจำเป็น
👤 เซลล์ (Sales) 📄 Quotation

ทำใบเสนอราคา & ส่งลูกค้า

เมื่อได้ Min แล้ว เซลล์จัดทำใบเสนอราคาผ่านระบบ (เลข Quotation รันอัตโนมัติตามเซลล์) แล้วส่ง PDF ทางเมล์ให้ลูกค้า พร้อมแนบแบบ (Drawing)

  • ระบบสร้างเลข Quotation อัตโนมัติ เช่น QGD-0326-KC001
  • บันทึกเป็น PDF
  • ส่งเมล์ผ่าน Outlook พร้อมแนบ PO, Drawing, Min
  • สามารถเพิ่มรูปภาพประกอบได้
🏢 ลูกค้า (Customer) ✅ Confirmation

ลูกค้ายืนยันใบเสนอราคา

เมื่อลูกค้าตกลง จะมี 2 ทางเลือก:

  • เปิด PO (Purchase Order) — ลูกค้าส่ง PO มาให้
  • Approve Quotation — ลูกค้าเซ็น Approval กลับมา
👤 เซลล์ (Sales) 📄 Work Order

เปิด Work Order (ปริ้น)

เซลล์เปิด Work Order ในระบบ เลข WO รันอัตโนมัติต่อเนื่อง (ไม่แบ่งตามเซลล์) เช่น WO26001

  • ระบุ Project, Location, Products, ทีมงาน
  • เลือกทีม: ส่งของอย่างเดียว / ทีมพื้น / ทีมโรงงาน 2 / ทีมติดตั้ง / ทีมประตู / ผู้รับเหมา
  • ระบุ QC Date, Installation Date
  • เอกสาร Checklist: PO, PR, Quotation, Min, Drawing Confirm, Waiting Confirm, Hand Over Job, Check List
👤 เซลล์ (Sales) 📄 Hand Over Job

เปิด Hand Over Job (ปริ้น)

เซลล์เปิดใบส่งมอบงาน เลข HO รันอัตโนมัติ เช่น HO26001

  • Project, Contractor, Location, Product
  • รายการสินค้าและจำนวน
  • แบบประเมินคุณภาพ (ผลิตภัณฑ์/ฝ่ายขาย/ฝ่ายช่าง)
👤 เซลล์ (Sales) 📄 Purchase Request

เปิด PR - กรณีสั่งของต่างประเทศ (ปริ้น)

เซลล์เปิดใบขอซื้อ เลข PR รันอัตโนมัติ เช่น PR26001

  • รายละเอียดสินค้า, Unit, QTY, Unit Price, Amount
  • Sub Total, VAT 7%, Net Total
👤 เซลล์ (Sales) 📎 รวมไฟล์

รวบรวมเอกสารเป็นไฟล์เดียว

เซลล์ปริ้นและรวมเอกสารทั้งหมดเป็นไฟล์เดียว:

  • PO (Purchase Order จากลูกค้า)
  • Min (Minimum Specification)
  • แบบ (Drawing)
  • Work Order
  • Hand Over Job
  • PR (ถ้ามี)

📤 กระบวนการจัดซื้อ

  • รับเอกสารที่ผ่านการอนุมัติครบทุกลำดับ
  • ตัดหน้าที่มีตัวเลขราคาขายออก (PO, Quotation, Min)
  • เลือกว่าจะส่งให้ทีมโรงงานคนไหนบ้าง
  • สแกนเอกสาร → ส่งเมล์ให้ทีมโรงงาน
  • ส่งเมล์ให้เจ้าของเซลล์เพื่อแจ้งสถานะ

🔙 กรณี Reject / แก้ไข

  • กรณี Work Order ผิด → ส่งกลับไปยังเซลล์เจ้าของเพื่อแก้ไข
  • เซลล์แก้ไขแล้วส่งเข้าสายอนุมัติใหม่
  • ระบบบันทึก Log การแก้ไขทุกครั้ง

💚 LINE แจ้งเตือนอัตโนมัติ

  • เมื่อเอกสารถูกส่งต่อ → ระบบส่ง LINE ถึงผู้อนุมัติคนถัดไปทันที
  • ข้อความแจ้งชื่อเอกสาร ชื่อโครงการ และ link Approve
  • ใช้ LINE Messaging API (Push Message) ผ่าน Backend
  • ผู้อนุมัติแต่ละคนต้องมี LINE User ID ที่ตั้งค่าในระบบ
  • จัดการ LINE User ID ได้ที่หน้า จัดการผู้ใช้ ในระบบ

ฟีเจอร์เพิ่มเติม (Additional Features)

🔙 ส่งกลับแก้ไข Work Order

กรณี Work Order ผิด สามารถกดส่งเอกสารกลับไปยังเซลล์เจ้าของงานเพื่อให้แก้ไข

✅ ปิดงาน Work Order

เมื่อช่างทำงานเสร็จ ทีมโรงงานเข้ามาปิด Work Order ว่างานนี้สำเร็จแล้ว

📊 กราฟสรุปรายเดือน

สรุปจำนวน Work Order ของเซลล์แต่ละคนออกมาเป็นกราฟในแต่ละเดือน

👤 แยก Work Order ตามเซลล์

Work Order จะต้องแบ่งตามเซลล์นั้นๆ เพื่อเวลาเรียกดูข้อมูลเดิม

⏰ แจ้งเตือน 3 เดือน

หาก Work Order ไหนที่ยังไม่มีการปิดงานภายใน 3 เดือน จากทีมโรงงาน ให้แจ้งเตือนและใส่ข้อมูลว่าทำไมถึงยังไม่ปิด

🖨️ ปริ้นเอกสารจากโรงงาน

เมื่อโรงงานได้รับเอกสาร สามารถปริ้นได้ เพื่อให้ช่างไปทำงาน

📎 Attach File ภายนอก

PO, Drawing, Min จะดึงมาจากไฟล์ข้างนอกโปรแกรม ต้องสามารถ Attach file ได้

🔔 Notification

แจ้งเตือนอัตโนมัติเมื่อเอกสารถูกส่งมาเพื่อรออนุมัติ หรือเมื่อมีสถานะเปลี่ยนแปลง

💚 LINE Messaging API

เมื่อเอกสารเลื่อนไปยังผู้อนุมัติคนถัดไป ระบบส่งข้อความ LINE แจ้งเตือนอัตโนมัติ พร้อม link ตรงไปยังหน้า Approve ผู้อนุมัติแต่ละคนต้องมี LINE User ID ที่ตั้งค่าในระบบ
