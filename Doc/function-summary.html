🔐 Auth & Login
Functionพารามิเตอร์คำอธิบาย
pickUser(u)u: username stringคลิก user chip บน login page → เติม input แล้ว doLogin() อัตโนมัติ
doLogin()—อ่าน username + password จาก input → ตรวจสอบกับ USERS → โหลด DB → แสดงหน้า App → navigate('dashboard')
doLogout()—ล้าง currentUser → ซ่อน App → แสดง Login page
🧭 Navigation
Functionพารามิเตอร์คำอธิบาย
navigate(page)page: string keyเปลี่ยนหน้าเนื้อหา — highlight menu, set title, breadcrumb, เรียก renderer function ตาม page
dashboard quotations workorders handover pr approvals reports report_sales report_approval users
updateBadges()—นับ pending approvals แล้วแสดง badge แดงบน sidebar เมนู "รออนุมัติ"
toggleSidebar()—เปิด/ปิด sidebar บน mobile (เพิ่ม class .open)
closeSidebar()—ปิด sidebar + ลบ overlay

FunctionReturnคำอธิบาย

📊 Dashboard
Functionคำอธิบาย
renderDashboard()แสดง: Welcome banner, Stat cards (Quotation/WO/HO/PR count), Quick action buttons, มูลค่า Quotation รวม, Work Order ล่าสุด, Activity timeline, ปุ่ม Reset Demo
📋 Quotation
Functionพารามิเตอร์คำอธิบาย
renderQuotations()—แสดงตารางรายการ quotations ทั้งหมด + search bar + filter chip (draft/pending/approved) + ปุ่มสร้างใหม่
filterQuotations(val)val: textกรองตาราง quotation ด้วยข้อความค้นหา (quoNo, customerName, project)
filterQuoStatus(el, status)el: chip element, status: stringกรองด้วย status chip — toggle active chip
openNewQuotation()—เปิด modal ฟอร์มสร้าง quotation ใหม่ (auto-gen เลขที่, รายการสินค้า, คำนวณ total)
addQuoItem()—เพิ่มแถวรายการสินค้าใน modal quotation
calcQuoItems()—คำนวณ amount ทุก row → update subTotal, VAT, grandTotal ใน modal
saveQuotation(submit)submit: Booleanvalidate → สร้าง quotation object → push ใส่ db.quotations → saveDB() ถ้า submit=true → status='pending', เพิ่ม approvalLog, แจ้ง LINE ไปยัง sales2
submitForApproval(docType, docId)docType: 'quotation', docId: stringเปลี่ยน draft → pending (step 1) → เพิ่ม log → แจ้ง LINE ไปยัง sales2
viewQuotation(id)id: stringเปิด modal แสดงรายละเอียด quotation + approval chain + ประวัติ log + ปุ่มพิมพ์ + ปุ่มสร้าง WO (ถ้า approved)
🔧 Work Order
Functionพารามิเตอร์คำอธิบาย
renderWorkOrders()—แสดงตาราง work orders ทั้งหมด + สถานะปิดงาน
openNewWorkOrder(quoId)quoId: string|undefinedเปิด modal ฟอร์มสร้าง WO ใหม่ ถ้ามี quoId จะ pre-fill ข้อมูลจาก quotation นั้น พร้อม drag & drop upload เอกสาร (PO, Drawing, MoM)
saveWorkOrder()—validate → สร้าง WO object พร้อม docFiles → push ใส่ db.workOrders → เพิ่ม approvalLog step 1 → saveDB()
viewWorkOrder(id)id: stringแสดง modal รายละเอียด WO + approval chain + ไฟล์แนบ + ประวัติ log + ปุ่มพิมพ์
closeWorkOrder(id)id: stringset isClosed=true + closedAt → saveDB()
🤝 Hand Over Job
Functionพารามิเตอร์คำอธิบาย
renderHandOver()—แสดงตาราง hand over jobs + คะแนนเฉลี่ย + ปุ่มให้คะแนน (ถ้ายังไม่ประเมิน)
openNewHandOver()—เปิด modal ฟอร์มสร้าง HO อ้างอิง Work Order ที่ approved แล้ว
saveHandOver()—validate → สร้าง HO object → push ใส่ db.handOverJobs → saveDB()
rateHandOver(id)id: stringเปิด modal ฟอร์มให้คะแนน 3 ด้าน (สินค้า, งานขาย, ติดตั้ง) + ความคิดเห็น
saveRating(id)id: stringบันทึกคะแนน qualityProduct/Sales/Install + comment → status='completed' → saveDB()
viewHandOver(id)id: stringแสดง modal รายละเอียด HO + คะแนน (ถ้ามี) + ปุ่มพิมพ์
🛒 Purchase Request
Functionพารามิเตอร์คำอธิบาย
renderPR()—แสดงตาราง purchase requests ทั้งหมด + มูลค่า + วันที่ต้องการ
openNewPR()—เปิด modal ฟอร์มสร้าง PR ใหม่ dropdown โครงการอิง quotations ที่มี + รายการสินค้า + upload เอกสาร
addPRItem()—เพิ่มแถวรายการใน modal PR
calcPRItems()—คำนวณ amount → update subTotal, VAT, netTotal ใน modal
savePR()—validate → สร้าง PR object → push ใส่ db.purchaseRequests → status='pending' → saveDB()
viewPR(id)id: stringแสดง modal รายละเอียด PR + รายการสินค้า + ยอดรวม + ปุ่มพิมพ์
✍️ Approvals — สายอนุมัติ
Functionพารามิเตอร์คำอธิบาย
renderApprovals()—แสดงเอกสารรออนุมัติของ currentUser + ประวัติ approval logs ทั้งหมด 20 รายการล่าสุด
approveDoc(docType, docId)docType: 'quotation'|'workorder', docId: stringเพิ่ม step, ตรวจว่าครบ step สุดท้าย → เปลี่ยน status='approved' → เพิ่ม approvalLog → saveDB() → แจ้ง LINE ผู้อนุมัติถัดไป
rejectDoc(docType, docId)docType, docIdเปิด modal ขอเหตุผล → เรียก confirmReject()
confirmReject(docType, docId, step)step: Numbervalidate เหตุผล → set status='rejected', step=0 → เพิ่ม log action='reject' → saveDB()
📈 Reports
Functionคำอธิบาย
renderReports()สร้าง report tab bar (3 แท็บ) + render placeholder sections
switchReportTab(tab, el)สลับ active แท็บและ section ที่แสดง
renderReportOverview()สรุปภาพรวม: Stat cards, stacked bar Quotation/WO, ตาราง summary ทุก document type
renderReportSales()วิเคราะห์ยอดขายตามลูกค้า: horizontal bar chart, top customer ranking, ตารางรายละเอียดโครงการทั้งหมดพร้อม WO/HO link
renderReportApproval()ประสิทธิภาพอนุมัติ: Stat cards (approve/reject/rate%), bar chart จำนวนต่อ step, bar chart per approver, vertical timeline, ตาราง summary ต่อเอกสาร
🖨️ Print
Functionพารามิเตอร์คำอธิบาย
printDocument(type, id)type: 'quotation'|'workorder'|'pr'|'handover', id: stringสร้าง HTML ต้นฉบับเอกสาร → เปิด popup window → print → ปิด window อัตโนมัติ
แต่ละประเภทมี template พร้อม:

- ตาราง detail + รายการสินค้า
- Signature block ที่ปรับตามประเภทเอกสาร
💚 LINE Messaging API
Functionพารามิเตอร์คำอธิบาย
renderUsers()—หน้า "จัดการผู้ใช้" — แสดงตารางผู้ใช้ทั้งหมด + LINE User ID ปัจจุบัน + ปุ่ม Edit/Test + ช่อง Channel Access Token
editUserLine(userId, username)userId: key ใน USERSเปิด modal กรอก LINE User ID (ต้องขึ้นต้นด้วย U และยาว 33 ตัวอักษร)
saveUserLine(userId)userId: key ใน USERSvalidate format → บันทึก lineUserId ลงใน USERS object (in-memory) → toast
testLineNotify(userId)userId: key ใน USERSส่ง test notification ไปยัง user ที่เลือก → เรียก simulateLINENotify()
simulateLINENotify(user, message)user: USERS object, message: stringแสดง popup สีเขียว simulate LINE message — แสดงชื่อผู้รับ, LINE user ID, สถานะ token, ข้อความ — หายหลัง 5 วินาที
getNextApprover(currentStepIndex)currentStepIndex: Number (0-based)หา role ของ step ถัดไป → หา user ใน USERS ที่มี role นั้น → return user object หรือ null
💡 หมายเหตุ: ในระบบ Demo นี้เป็นเพียงการ simulate การส่ง LINE — ไม่ได้ call API จริง การใช้งานจริงต้องมี Backend server ที่ call POST https://api.line.me/v2/bot/message/push พร้อม Channel Access Token และ lineUserId ของแต่ละ user
📎 File Upload
Functionพารามิเตอร์คำอธิบาย
handleDragOver(event)event: DragEventpreventDefault → เพิ่ม CSS class drag-over บน drop zone
handleDragLeave(event)event: DragEventลบ class drag-over
handleDrop(event, key)key: 'po'|'drawing'|'min'|'pr_doc'รับ files จาก dataTransfer → เรียก processFiles()
handleFileSelect(input, key)input: file input element, key: stringรับ files จาก input → เรียก processFiles()
processFiles(files, key)files: FileList, key: stringpush file metadata {name, size, type} เข้า uploadedFiles[key] → render file list badges
getFileIcon(filename)filename: stringreturn emoji icon ตาม extension (.pdf→📄, .jpg→🖼️, .doc→📝, .dwg→📐 ฯลฯ)
formatFileSize(bytes)bytes: Numberแปลง bytes → KB / MB string

ไฟล์ถูกเก็บใน variable uploadedFiles (in-memory) ระหว่างสร้าง Work Order — metadata (ชื่อ, ขนาด) ถูกบันทึกใน workOrders[].docFiles แต่ไม่มีการ upload file จริงในระบบ Demo

💾 Database Helpers
Functionคำอธิบาย
loadDB()อ่าน JSON จาก localStorage['greendii_demo_db'] → parse → ใส่ตัวแปร db ถ้าไม่มีข้อมูล → เรียก seedData()
saveDB()Stringify db object → บันทึกกลับลง localStorage
resetDB()ลบ localStorage key → เรียก seedData() → navigate dashboard → toast
seedData()สร้างข้อมูลตัวอย่าง (3 quotations, 2 work orders, 1 hand over, 1 PR, approval logs, notifications) → saveDB()
